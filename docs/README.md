# audioBlade Documentation Map

## Canonical Documents

- `TECHNICAL_PLAN.md`: technical rationale, platform choices, references, and architecture.
- `DEVELOPMENT_GUIDELINES.md`: engineering rules and scope control.
- `DEVELOPMENT_PROCESS.md`: polyglot development process and toolchain ownership.
- `Plan/`: canonical English planning documents.
- `Harness/`: Harness Engineering role and workflow notes.
  - `Harness/agents.md`: role index and project-specific agent map.
  - `Harness/workflow.md`: project-level Harness workflow, task types, and artifact rules.
  - `Harness/team.md`: active team charter, role responsibilities, and escalation rules.

## Chinese Readability Copies

- `cn/`: Chinese translations or readability copies.
- `cn/Plan/`: Chinese translations of canonical English plans.

Planning documents should be authored in English under `docs/Plan/` first. Chinese versions should mirror them under `docs/cn/Plan/` when needed.
